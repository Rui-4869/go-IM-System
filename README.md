# go-IM-System

简单的基于 TCP 的即时通讯（IM）示例，使用 Go 实现。包含一个可运行的服务器端和客户端示例，用于本地局域网或本机学习与测试。

**特性**
- 支持广播（公聊）消息
- 支持私聊（指定用户名发送）
- 支持查询在线用户列表
- 支持更新用户名
- 基于简单的文本协议，便于理解与扩展

**主要文件**
- [main.go](main.go) — 启动服务器的入口（创建并启动 `Server`）
- [server.go](server.go) — 服务器实现：连接管理、消息广播、超时踢人
- [user.go](user.go) — `User` 对象实现：上线/下线、消息处理、私聊与重命名逻辑
- [client.go](client.go) — 简单交互式客户端：菜单、命令行参数、发送/接收逻辑
- [go.mod](go.mod) — Go 模块文件

**协议 / 命令说明**
- 广播消息：直接输入文本并回车，服务器会广播给所有在线用户
- 查询在线用户：客户端向服务器发送 `who`（客户端里通过 SelectUsers 实现）
- 更新用户名：客户端发送 `rename|<用户名>`，例如 `rename|张三`
- 私聊消息：客户端发送 `to|<用户名>|<消息内容>`，例如 `to|张三|你好` 

**运行说明**
1. 启动服务器（在一台机器或终端）——默认监听 `127.0.0.1:8888`：

```bash
# 在包含 main.go, server.go, user.go 的目录运行：
go run main.go server.go user.go
```

> 注意：不要使用 `go run .`，因为仓库同时包含客户端与服务端的 main 函数，可能导致冲突。请显式运行服务端需要的文件。

2. 启动客户端（在另一终端或另一台机器）：

```bash
# 指定服务器 IP 与端口（可选），默认：127.0.0.1:8888
go run client.go -ip 127.0.0.1 -port 8888
```

3. 客户端交互
- 启动后会显示菜单：`1.公聊模式`、`2.私聊模式`、`3.更新用户名`、`0.退出`。
- 公聊：选择 `1` 后输入文本并回车；输入 `exit` 返回菜单。
- 私聊：选择 `2`，先输入目标用户名，再输入消息（格式客户端会封装为 `to|<用户名>|<消息>`）。
- 更新用户名：选择 `3`，输入新用户名，客户端会发送 `rename|<用户名>`。

**实现要点**
- 服务端通过 `Server.Message` channel 广播消息给所有在线用户。
- 每个 `User` 有一个单独的 `C chan string`，负责接收来自广播的消息并写回连接。
- 服务端为每个连接启动读协程并通过 `isLive` 通道实现 300 秒无活动则踢出的超时机制。
- 客户端使用 `io.Copy(os.Stdout, conn)` 在后台 goroutine 打印来自服务端的实时消息。

**扩展建议**
- 增加消息格式化（时间戳、JSON）以便前端解析
- 使用 TLS 加密连接
- 引入认证/持久化用户数据
- 支持群聊房间（channel/room）概念

**已知/注意事项**
- 本示例为学习/演示用途，未做生产级别的错误处理、安全或并发细节加强。
- 在同一目录下不要同时 `go run .`（会包含客户端与服务端 main），请按示例分别启动。
